## 개요#









## docker #

- build
- ship
- run



## Build #

### 이미지 파일 재료 ##

- Dockerfile
- context

### Dockerfile 구조  ##
> FROM COPY ENTRYPOINT

1. FROM
2. ENV
3. RUN
4. COPY
5. ENTRYPOINT [" "]



### Dockerfile 이미지 빌드 ##

```bash
docker build -t <이미지 이름> ./디렉터리 # -t 옵션은 이미지이름:태그 지정
```



## Run #


```bash 
docker run --name 컨테이너이름 이미지이름
```


### docker run 옵션 ##

1. 바인드 마운트
```bash 
-v /tmp/폴더/:/mnt/:ro 
```

2. 볼륨
```bash 
-v shared-vol:/mnt/:ro 

#볼륨 표시/ 삭제
docker volume ls
docker volume rm shared-vol
```


3. 컨테이너 포트 공개
```bash 
-p 127.0.0.1:8080:80
```


3. 컨테이너 대화식 실행
```bash 
-it 이미지 /bin/bash
```

### 컨테이너 컨테이너 종료 및 삭제 ##
```bash 
docker stop 컨테이너
docker rm 컨테이너
```




### 컨테이너 내부 명령어 실행 (exec) ##

```bash
docker exec -it 컨테이너이름 /bin/bash # -it 옵션 : 대화식으로 셸 조작
```








## 도커 컴포즈#

```bash
docker compose build #컴포즈 파일에 지정한 빌드 실행
docker compose up # 컴포즈 파일 컨테이너 가동
docker compose down # 컴포즞 파일에 컨테이너 정지 후 삭제
```
